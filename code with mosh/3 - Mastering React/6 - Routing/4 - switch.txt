The Route component matches what is on a given URL with the value that is set on its path attribute, and every succesfull match will be render its component, to solve this messy problem, there are 2 solutions:

    1.- add to each Route component after the path attribute the word exact... of this way, it'll be rendered just the component that its path value macthes exactly to  the URL...

    2.- Import the Switch component from react-router-dom and wrapp all Route components inside a Switch component....
        This Switch component will rendered the first child that matches its path value to the given URL....
        That is why, if we are following this approach, we should order the Route components from the most specific to the most genericall...
